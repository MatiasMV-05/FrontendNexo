## **PROYECTO FINAL** 

## **TECNOLOGÍAS WEB II (SIS-215)** 

## **1. TÍTULO** 

**Desarrollo de una Aplicación Web Frontend Moderna utilizando React, TypeScript, JWT y Consumo de APIs REST** 

## **2. OBJETIVO** 

Desarrollar una aplicación web frontend funcional que integre los conocimientos adquiridos durante la asignatura, aplicando tecnologías modernas de desarrollo frontend, consumo de servicios web, autenticación mediante JWT, control de acceso por roles, diseño responsivo, accesibilidad, SEO y despliegue en la nube. 

## **3. MODALIDAD Y FECHA DE ENTREGA** 

- Individual 

- Defensa obligatoria. 

- Fecha de entrega y defensa 

## **4. DESCRIPCIÓN DEL PROYECTO** 

Cada estudiante deberá desarrollar una aplicación web utilizando React y TypeScript que consuma una API REST. 

La temática del sistema será libre **pero única por cada estudiante.** 

Ejemplos: 

- Sistema de Inventario 

- Biblioteca Virtual 

- Gestión de Cursos 

- Sistema de Eventos 

- Catálogo de Productos 

- Gestión de Películas 

- Sistema de Noticias 

- Gestión de Tareas 

- Sistema de Reservas 

- Otro a elección del estudiante 

La aplicación deberá demostrar la integración de todos los contenidos desarrollados durante la asignatura.

## **5. REQUERIMIENTOS TÉCNICOS OBLIGATORIOS** 

## **5.1 Tecnologías requeridas** 

La aplicación deberá desarrollarse utilizando: 

- React 

- TypeScript 

- React Router 

- Consumo de APIs REST mediante Fetch o Axios 

- Git y GitHub 

- Otras librerías que el estudiante requiera 

## **5.2 Arquitectura del Proyecto** 

La aplicación deberá presentar una estructura organizada basada en componentes. 

Como mínimo deberá existir: 

- components 

- pages 

- services 

- routes 

- interfaces o types 

- assets 

Se evaluará la organización y reutilización de componentes. 

## **5.3 Navegación** 

La aplicación deberá contener al menos: 

- Landing page del rubro 

- Página de Login 

- Página de Crud Principal 

- Página 404 

## **5.4 Base de Datos y Backend** 

La aplicación deberá consumir una API pública disponible en Internet no necesariamente relacionada con la temática seleccionada. 

## **Puntaje Extra (Opcional)** 

El estudiante podrá desarrollar e integrar una API propia para complementar o reemplazar parcialmente el consumo de la API pública. 

## **5.5 Autenticación** 

La aplicación deberá implementar un sistema de autenticación. 

Podrá realizarse mediante: 

## **Opción A (Obligatoria)** 

Autenticación simulada en frontend utilizando: 

LocalStorage o SessionStorage. 

Usuarios predefinidos. 

Context API. 

Protected Routes. 

## **Opción B (Puntaje Extra)** 

Autenticación real mediante JWT utilizando una API propia. 

La autenticación deberá permitir: 

Inicio de sesión. 

Persistencia de sesión. 

Cierre de sesión. 

Protección de rutas privadas. 

## **5.6 Roles de Usuario** 

El sistema deberá implementar obligatoriamente dos roles. 

## **Administrador** 

Permisos: 

- Iniciar sesión. 

- Visualizar registros. 

- Crear registros. 

- Modificar registros. 

- Eliminar registros. 

## **Usuario** 

Permisos: 

- Iniciar sesión. 

- Visualizar registros. 

Restricciones: 

- No podrá crear registros. 

- No podrá modificar registros. 

- No podrá eliminar registros. 

- No podrá administrar usuarios. 

La diferencia de permisos deberá evidenciarse visualmente mediante rutas protegidas, menús o componentes restringidos. 

## **5.7 Formularios y Validaciones** 

La aplicación deberá incluir como mínimo: 

- Formulario de inicio de sesión. 

- Formularios CRUD. 

Se deberá validar: 

- Campos obligatorios. 

- Longitud mínima de texto. 

- Formato de ingreso de valores. 

- Mensajes de error visibles al usuario. 

## **5.8 Diseño Responsivo** 

La aplicación deberá adaptarse correctamente a: 

- Dispositivos móviles. 

- Tablets. 

- Computadoras de escritorio. 

## **5.9 UX/UI y Accesibilidad** 

Aplicar: 

- Navegación intuitiva. 

- Diseño consistente. 

- Mensajes de éxito y error. 

- Etiquetas label. 

- Texto alternativo en imágenes. 

- Contraste adecuado. 

## **5.10 Seguridad Básica** 

Implementar: 

- Protección de rutas. 

- Validación de entradas. 

- Manejo de errores. 

- Restricción de acciones según rol. 

## **5.11 SEO** 

La aplicación deberá incorporar: 

- Título dinámico por página. 

- Meta Description. 

- Open Graph básico. 

- HTML semántico. 

- Imágenes con atributo ALT. 

## **5.12 Despliegue en la Nube** 

La aplicación deberá estar publicada en: 

- Vercel u otro 

La URL deberá encontrarse operativa durante la defensa. 

## **5.13 Uso de Plantillas React (Opcional)** 

Se permite utilizar plantillas profesionales React. 

Ejemplos: 

- Material Dashboard 

- CoreUI 

- AdminLTE React 

- Argon Dashboard 

- Material UI Templates
